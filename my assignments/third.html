2. Database Structure Analysis and Improvements

Improving the Database Structure

To enhance the efficiency and accuracy of the virtual library database, the following improvements were implemented:

Normalization

The data was reorganized into three interconnected tables:

  • Books: Includes details of books, loan status, and borrower information
  • Authors: Stores unique author records with birthdates and IDs
  • Reviews: Stores user-submitted feedback per book

Key Structural Improvements

  1. Author Normalization: Created a separate Authors table with unique author_ID values to prevent redundancy
  2. Review Management: Implemented a dedicated Reviews table with ratings and comments linked to specific books
  3. Loan Status Tracking: Enhanced the Books table with available status, checkout dates, and return dates
  4. Data Quality Issues: Identified placeholder values (########) that need proper formatting or validation
  5. Genre Classification: Books are categorized into clear genre groups for better organization

3. Problems I Encountered

Main Challenges

While creating this database, I ran into a few basic issues:

  • Boolean Format Problems: I used "Yes/No" instead of "TRUE/FALSE" which made the validation fail. My professor gave me a hint to explore data types and check which would fit in the available section, then I realized I needed to use proper boolean values. Once I fixed this, everything worked.
  • Simple Naming Issues: I had "Book_ID" in one table but "book_ID" in another. This small difference broke the connections between tables.
  • Text Encoding: Special characters like apostrophes showed up weird in the CSV files, making some book titles look messy.

These may seem like small problems, but they prevented the whole system from working properly until fixed!

4. My Virtual Library Concept

Book Knowledge Verification System

If I owned a virtual library, I would implement a unique verification system:

  • First-time customers would be exempt from verification
  • Returning borrowers would need to enter their name and ID at the entrance
  • The system would identify books they've borrowed previously
  • To gain entry, they would need to answer one question about a book they borrowed
  • If they fail to answer correctly, library access would be denied until they provide the correct answer

This approach would encourage readers to actually engage with the books they borrow rather than simply collecting them. It would create a community of thoughtful readers who appreciate the content of what they read!

Benefits of This System

  • Encourages more mindful and attentive reading
  • Reduces the number of books borrowed but never read
  • Creates a fun challenge for library users
  • Helps track user engagement with different types of books
  • Makes the virtual library experience more interactive

6. Recommendations for Future Database Enhancements

  3. Genre Standardization: Create a separate Genres table to standardize genre classifications
  4. Data Validation: Implement proper date formats and validation rules
  5. Inventory Management: Add fields for tracking book conditions, acquisitions, and replacements
